## 1. 总体架构与主界面布局设计

系统采用 `PyQt5` 作为基础 UI 框架，`Matplotlib` 作为嵌入式图形渲染引擎，采用经典的“菜单导航 + 工作区 + 日志区”的三段式结构。为保证中文显示的规范性，全局字体设定为“微软雅黑”（Microsoft YaHei）。

总体布局（`MainWindow`）分为以下三个核心区域：
1. **顶部导航栏（Menu Bar）**：放弃传统的下拉菜单嵌套结构，采用平铺式的菜单按钮（模块0至模块3），用户点击后可快速无缝切换至对应的工作流模块。
2. **中部核心工作区（Stacked Widget）**：占据屏幕约 70% 的高度。采用堆叠窗口容器（`QStackedWidget`）管理各个模块的子界面，确保每次只显示当前激活的模块。
3. **底部日志输出区（Log Browser）**：占据屏幕约 30% 的高度。采用暗黑风格（背景 `#2b2b2b`）的富文本浏览器，通过重写 Python 的 `logging.Handler` 实现后台日志的线程安全重定向，实时打印算法运行进度、警告及异常信息。

## 2. 基础模块视图规范 (BaseModuleUI)

为保持软件交互的一致性，提取了通用界面的基类。除模块0外，其余模块子界面的布局严格遵循 **左侧面板（占位1/3）** 与 **右侧预览（占位2/3）** 的分割比例：

*   **左侧控制面板**（自上而下）：
    *   **数据输入 (Input)**：提供原始数据或前序中间结果的文件路径选择（含“浏览”按钮）。
    *   **数据输出 (Output)**：提供当前模块运行成果的默认存储路径（支持自定义更改）。
    *   **参数设置 (Parameters)**：提供核心算法的参数输入框（如步长、阈值）。
    *   **执行按钮 (Run Button)**：绿色高亮的大号按钮，点击后触发后台异步线程（`QThread`）执行核心算法，防止主界面卡死。
*   **右侧成果预览 (Preview)**：
    *   内嵌 `FigureCanvas`，用于绘制 Matplotlib 图表。
    *   统一配备名为 **“加载数据并刷新预览视图”** 的按钮，执行完成后也可点击刷新最新成果。
    *   所有子图表的图例严格统一放置于 **右下角 (lower right)**，字号设为 `10`。
    *   运行结束提示窗标准格式：`moduleX: 模块名 已完成！`。

## 3. 各模块子界面详细设计

### 3.1 模块0：数据预处理界面 (Module0UI)
*   **数据输入**：河岸线 (.shp)、实测河岸点 (.shp)、剖面点 (.shp)。
*   **数据输出**：清洗后的河岸线、左/右河岸点、标准化剖面点。
*   **参数设置 & 成果预览**：此模块仅做数据格式化与拓扑判别，因此隐藏参数设置与可视化预览面板，保持界面简洁。

### 3.2 模块1：深泓点内插界面 (Module1UI)
*   **数据输入**：河岸线、剖面点。
*   **数据输出**：河道轴线、深泓线、深泓点、深泓点内插。
*   **参数设置**：`步长 (m)`，默认值 `5.0`。
*   **成果预览**（包含上下两个子图表）：
    1.  **平面展示**：以真实的 X、Y 坐标为基础，绘制蓝色的“深泓线”与红色的实测“深泓点”。
    2.  **高程展示**：将弯曲的深泓线一维化（拉直）。横轴为累积投影里程 `S`，纵轴为高程 `Z`。以平滑的绿色曲线展示“深泓点内插”，红色散点叠加展示原始“深泓点”位置。

### 3.3 模块2：河岸点内插界面 (Module2UI)
*   **数据输入**：河岸线、深泓线、深泓点内插、左右实测河岸点。
*   **数据输出**：左河岸点内插、右河岸点内插。
*   **参数设置**：`阈值 (m)`，默认值 `300.0`（用于控制法向量求交的最大搜索范围）。
*   **成果预览**（引入选项卡 `QTabWidget` 控制）：
    *   由于需要分别检视左右岸，提供 **“左河岸预览”** 与 **“右河岸预览”** 两个选项卡切换。
    *   每个选项卡同样包含“平面展示”与“高程展示”两张子图。
    *   **防跳变机制（重点设计）**：为防止切换选项卡时因左右岸尺寸不一导致的图表缩放、位移跳变，底层算法提取了河岸线的**全局物理边界 (Total Bounds)**、全局高程极值以及全局投影里程最大值。左右岸的绘图区（Bounding Box）、刻度范围（X/Y/Z Limits）以及图表边距被强制锁定为严格一致的参数。

### 3.4 模块3：剖面点内插界面 (Module3UI)
*   **数据输入**：深泓线、深泓/左/右岸点内插、原始实测剖面点。
*   **数据输出**：高密度三维点云——剖面点内插 (.shp)。
*   **参数设置**：`步长 (m)`，默认值 `2.0`。
*   **成果预览**（精细剖面对比图）：
    *   **控件交互**：在顶部提供下拉选择框（`QComboBox`），自动读取生成成果的 ID 前缀（如 `001_002_003`），用户可切换检视任意内插剖面。
    *   **对齐视图逻辑**：提取选中 ID 所关联的上游实测剖面（如 001）、下游实测剖面（如 002）以及当前内插剖面。将三条剖面的 **“最低深泓点” 作为零点中轴线对齐**（即 X=0）。
    *   **渲染设定**：
        *   横轴：相对横向距离（中心深泓点为0，左负右正）。纵轴：绝对高程 Z。
        *   绘制中轴垂直辅助线，便于查看非对称河床的演变。
        *   实测剖面使用虚线/点划线，内插剖面使用红色实线强调。用于直观检验形态融合（Morphing）的高保真度。
