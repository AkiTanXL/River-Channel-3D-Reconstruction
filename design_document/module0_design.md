## 处理过程：

### 1. 河岸线拓扑识别与属性重构：
读取原始的 **河岸线.shp**（需包含两条折线）。程序首先利用实测剖面点推算出河流的整体流向，随后基于流向和空间拓扑关系，自动判别出左岸线与右岸线。最后重新输出为新的 shp 文件，并追加 `id` 字段进行标识：左岸线 `id=0`，右岸线 `id=1`。

### 2. 河道边界面生成：
在识别出左右岸线的基础上，提取两条线的坐标节点构建闭合多边形，并输出为 **河道边界.shp**（`id=1`），为后续的网格划分或可视化提供完整的面状边界范围。

### 3. 河岸点归类与一维化排序：
读取 **实测河岸点.shp**（需由用户指定包含 x, y, z 信息的字段名）。程序计算每个散点到左右岸线的距离，将其自动归类为左岸点或右岸点。为保证数据的一维单调性，程序统一从最东北方向的岸线端点作为起算点，将所有点投影到岸线上并按投影里程进行排序。分别输出为 **左河岸点.shp** 和 **右河岸点.shp**，均包含 `x, y, z, id` 四个字段，`id` 字段采用 `00i` 格式（表示沿流向的第 i 个点）。

### 4. 剖面点清洗与编码标准化：
读取 **剖面点.shp**。利用用户指定的属性字段读取空间坐标和原始 ID 标识。程序通过正则表达式提取原始 ID 中的数字作为剖面组号，并按照该剖面组内部的顺序重新编码。输出新的散点文件，`id` 字段统一规范为 `00i_00j` 格式，其中 `i` 代表剖面编号，`j` 代表该点在此剖面中的点号。

## 核心算法：

### 1. 剖面质心拟合与流向推算算法
由于原始河岸线缺乏方向性，算法首先根据原始剖面点中提取的编号 `i` 进行分组，计算每个实测剖面的中心点（几何质心）。通过连接编号最小（首）与最大（尾）的剖面中心点，生成一个贯穿全局的“平均流向向量”（Flow Direction Vector）。

### 2. 左右岸智能拓扑判别算法
获取“流向向量”后，将其逆时针旋转 90 度，得到指向河流左侧的“左侧法向量”。分别计算两条河岸线的几何中心点，并求出其相对于河流绝对中心（两线中心的中点）的偏移向量。通过计算偏移向量与“左侧法向量”的点积（Dot Product），点积大于0的即为左岸，从而实现全自动的左右岸判别。

### 3. 曲线投影寻址与排序算法
为解决河岸点可能存在的乱序问题，算法强制将左右岸线的起点统一朝向东北方向（即坐标和 `X+Y` 最大端）。随后调用 `Shapely` 的 `project` 函数，将每个河岸点正交投影至对应的岸线上获取其“一维里程（S）”。基于该里程对散点进行升序排列，确保后续内插的空间拓扑连贯性。

### 4. 河道面拓扑闭合与方向一致性校验算法
为避免左右岸线数字化绘制方向不一致（如一侧从上游画、另一侧从下游画）导致生成面时出现“交叉错位（领结效应）”，算法在拼接坐标前会进行方向校验：
分别计算“左岸起点到右岸起点”与“左岸起点到右岸终点”的欧氏距离。若前者大于后者，说明两线绘制方向相反。此时程序会自动翻转右岸的节点顺序，随后再按照“左岸正向 + 右岸逆向”的顺序拼接并强制首尾闭合，从而确保生成的河道多边形拓扑完全正确。
